Your coding agents are capable but context-starved.
Kiln fixes that.

Kiln is an open-source, AI-native SDLC tool. It keeps one knowledge graph linking intent → requirement → blueprint → work order — and hands agents fully-contextualized work orders over MCP.

Free & open source · macOS & Windows

First launch on macOS: Kiln isn’t notarized with Apple yet, so right-click the app and choose Open the first time. After that it opens normally and updates itself.

Upstream of your coding agents

Kiln is deliberately not a coding tool. It makes sure that when an agent picks up a unit of work, it receives the complete chain of “what” and “why” behind it — in a single call.

⛓

One knowledge graph

Product intent, requirements, technical blueprints, and work orders live in one linked graph — not scattered across a doc, a tracker, and a transcript.

🤝

Agent handoff over MCP

Claude Code, Cursor, or any MCP client pulls a ready work order and receives the full intent → requirement → blueprint context in one call, then reports status back.

✍️

Agent-assisted authoring

Draft, extract, refine, and review agents help you author the chain — every suggestion behind explicit accept/reject gates, with revision history.

🩻

X-ray & Pulse

See the whole graph, its readiness and gaps, on one map — and a home dashboard that tells you what’s ready, what’s blocked, and what drifted.

🔍

Context Inspector

Inspect the exact context an agent is handed for any work order, with a receipt recorded on every delivery. No more guessing what the agent knew.

🗂

Local-first projects

Each project is one SQLite file on your machine. Your specs, your key, your data — with markdown export whenever you want out.

How it works

From fuzzy intent to an agent shipping code, without losing the thread.

Capture intent

Start from the “why”. Author it yourself or let the draft agent turn a conversation into structured requirements.

Blueprint the how

Each requirement gets a technical blueprint. Refine and review agents propose changes; you accept or reject every one.

Cut work orders

Break blueprints into ready, self-contained work orders on the Board — each one linked back to its blueprint, requirement, and root intent.

Hand off to an agent

Point any MCP client at Kiln. The agent calls get_work_order, receives the full lineage in one payload, does the work, and reports status back.
